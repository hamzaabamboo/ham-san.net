# Localization Quality

The English, Japanese, and Thai experiences must all feel intentional and native to the design system.

## Acceptance Criteria

- Japanese text avoids awkward register, placeholder phrasing, and untranslated structural jargon
- Thai text avoids broken rhythm, clumsy transliteration, and UI text that feels machine-generated
- English text stays concise, precise, and free of invented brand fluff not present in the Stitch comps
- Mixed-language CMS archive content is allowed when it is the real source record, but the page must explicitly frame it as source/archive content
- Locale-specific typography supports legibility and visual parity without breaking layout density
- Each supported locale preserves the same information architecture and navigational cues
